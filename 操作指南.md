# WiseStar-MathAgent 本地运行说明（简化版）

本 README 只关注「本地启动前后端 Web 界面」这一件事，便于在自己机器上快速复现。

---

## 1. 目录结构（和本地启动相关的部分）

- `backend/`：FastAPI 后端服务
- `frontend/`：React + webpack 前端
- `code/`、`utils/`、`data/`、`output/`：算法与数据模块（命令行/脚本使用时才需要）
- `.env`：项目根目录的环境变量文件（后端会读取）

---

## 2. 环境要求

- Python ≥ 3.10（建议 3.11）
- Node.js ≥ 18（安装时勾选：
  - `Node.js runtime`
  - `npm package manager`
  - `Add to PATH`）
- 操作系统：Windows / macOS / Linux 均可

后端依赖通过 `backend/requirements.txt` 安装；根目录 `requirements.txt` 主要用于算法脚本。

---

## 3. 配置环境变量

在项目根目录新建 `.env` 文件，例如：

```bash
DEEPSEEK_API_KEY=你的_api_key
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat
MAX_RETRY=3
TIMEOUT=30
````

具体字段以 `backend/config.py` 的定义为准；这里列出常用项。
后端统一从根目录 `.env` 读取配置。

---

## 4. 后端启动（推荐方式：虚拟环境 + uvicorn）

在项目根目录执行：

```bash
# 1. 创建并激活虚拟环境（只做一次）
python -m venv .venv
.\.venv\Scripts\activate      # Windows
# source .venv/bin/activate   # macOS / Linux

# 2. 安装后端依赖
pip install -r backend/requirements.txt

# 3. 启动后端（在项目根目录）
uvicorn backend.app:app --reload --host 0.0.0.0 --port 8000
```

启动成功后，用浏览器或命令行访问：

```bash
curl http://localhost:8000
```

如返回：

```json
{"message": "WiseStar-MathAgent API Server", "version": "1.0.0"}
```

说明后端正常。

> 说明：`backend/requirements.txt` 中已固定 `openai==1.10.0` 等版本，用于避免 HTTP 客户端不兼容问题。

---

## 5. 前端启动

1. 安装依赖（只做一次）：

   ```bash
   cd frontend
   npm install
   ```

2. 启动开发服务器：

   ```bash
   npm run dev
   ```

   看到日志中类似：

   ```text
   [webpack-dev-server] Project is running at:
   [webpack-dev-server] Loopback: http://localhost:3048/
   ```

   表示前端已启动成功。

3. 浏览器访问：

   ```text
   http://localhost:3048/#/dashboard
   ```

如果能看到页面而不是空白，说明前端构建正常。
前端调用后端的地址在 `frontend/src/utils/api.js` 中配置，保持：

```js
const API_BASE_URL = "http://localhost:8000";
```

即可与本地后端联通；若后端端口或域名有变化，只需修改这一行。

---

## 6. 简要排错指南

* 访问 `http://localhost:3048/#/dashboard` 只有白屏：

  * 打开浏览器开发者工具（F12），确认控制台没有模块加载错误；
  * 确认 `npm run dev` 仍在运行，且 `API_BASE_URL` 指向 `http://localhost:8000`。
* `curl http://localhost:8000` 无响应或报错：

  * 检查后端终端是否还在运行；
  * 检查 `.env` 是否填好，虚拟环境是否激活，并确认依赖已按 `backend/requirements.txt` 安装。
